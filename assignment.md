# 1일차 과제 명세서

## 주제: 바이브 코딩 & 컨텍스트 엔지니어링 실습 확장

## 1. 과제 목적

이번 과제의 목적은 결과물을 잘 만드는 것이 아니라, **AI에게 일을 시킬 때의 사고방식 변화를 경험하고 정리하는 것**입니다.

1일차 실습에서 경험한 다음 흐름을 개인 환경에서 다시 재현하는 것이 핵심 목표입니다.

- 요구만 던졌을 때 결과가 흔들렸던 경험
- 컨텍스트를 주자 결과가 달라졌던 경험

## 2. 과제 개요

아래 3가지 항목을 모두 수행해야 합니다.

1. Streamlit 기반 웹 페이지 3개 이상 구성
2. 바이브 코딩 → 컨텍스트 엔지니어링 과정 기록
3. 컨텍스트 전/후 결과 비교 및 분석

## 3. 과제 상세 요구사항

### 3.1 웹 페이지 구성

Streamlit으로 실행 가능한 웹 페이지를 최소 3개 이상 구성합니다.

**구현 방식은 자유입니다.**
- 하나의 app.py에서 페이지 분기
- 또는 섹션 단위로 구성

**각 페이지에는 최소 아래 요소가 포함되어야 합니다.**
- 페이지 제목 또는 서비스 이름
- 한 줄 설명
- 본문 영역(텍스트, 리스트, 섹션 등)
- 버튼 1개 이상

**디자인 완성도, UI 퀄리티는 평가하지 않습니다.** 화면에 실제로 요소가 나타나고 실행되는지만 중요합니다.

### 3.2 바이브 코딩 → 컨텍스트 엔지니어링 과정 기록

아래 두 단계를 모두 수행하고 기록해야 합니다.

#### (1) 컨텍스트 없는 요청
- 요구사항만 포함된 요청으로 AI에게 작업을 시킵니다.
- 예시:
  - 이 페이지를 더 좋게 만들어주세요
  - UI를 개선해주세요
  - 설명을 더 잘 보이게 바꿔주세요

#### (2) 컨텍스트를 추가한 요청
아래 4가지 요소를 포함하여 컨텍스트를 구조적으로 작성합니다.

- **목표(Goal)**: 이번 작업에서 무엇이 성공인지
- **현재 상태(Current State)**: 지금 페이지가 어떤 상태인지
- **제약 조건(Constraints)**: 하지 말아야 할 것, 우선순위
- **역할/관점(Role)**: 어떤 입장에서 판단해야 하는지

### 3.3 결과 비교 및 분석

보고서에 반드시 아래 내용을 포함해야 합니다.

- 컨텍스트 없는 요청의 결과 요약
- 컨텍스트를 추가했을 때 달라진 점
- 무엇이 더 명확해졌는지
- AI의 판단이 어떻게 달라졌다고 느꼈는지

**정답은 없으며, 개인의 관찰과 해석을 중시합니다.**

## 4. 제출물 구성

아래 2가지를 모두 제출합니다.

### 4.1 코드
- 실행 가능한 Streamlit 코드
- 실행 방법이 명확해야 합니다
- 제출 방식: GitHub 저장소에 업로드

### 4.2 보고서
- 형식: 자유(PDF 또는 Markdown 권장)
- 분량 제한 없음
- 필수 포함 항목:
  1. 사용한 환경 및 도구
  2. 구현한 웹 페이지 개요
  3. 컨텍스트 없는 요청 내용
  4. 컨텍스트를 추가한 요청 내용
  5. 결과 비교(전/후)
  6. 느낀 점 및 배운 점
- 스크린샷 포함은 권장사항이며 필수는 아닙니다.

## 5. 평가 기준

- 과제 요구사항 충족 여부
- 컨텍스트 4요소의 명확성
- 과정 기록의 충실도
- 결과 차이에 대한 이해 및 설명

**다음 항목은 평가하지 않습니다.**
- 디자인 퀄리티
- 코드 난이도
- 기술적 완성도
- 오류 발생 여부

## 6. 유의 사항

- 완성도가 낮아도 감점 요소가 아닙니다.
- 실패한 시도, 잘 안 된 결과도 문제없습니다.
- **중요한 것은 과정과 사고 방식의 변화입니다.**
